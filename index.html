May 21, 2014, Mentoring

Recently I decided to level up in my career so I signed up for a month of mentoring with Thoughtbot. So far this seems to have been a wise decision, although I have yet to actually meet with my mentor or get any real personalized feedback from the mentoring team.

My goals for this endeavor are as such:

  1. better understanding and mastery of Rails
  2. feel more confident when using a TDD approach that I'm writing the right tests
  3. make the transition from writing code that just works to writing code that is well designed
  4. return to my VI roots and start using VIM!
  5. transfer some of my visual artistic skills to pixels by learning web design fundamentals
Signing up for mentoring coincided with a major overhaul of my artist web site, so I began with the "Design for Developers" course. This gave me some ideas, although I have yet to really formalize the process with a bunch of sketches and more experimentation with layout.

The site currently has over 250 works of art on it, all images of different sizes, so it's kind of a monster. Getting all of that into a nicely formatted grid that is responsive and has appropriate white space around it is a challenge. So, work in progress. My goal was a site that felt like a coffee table book, that invited exploration and felt unstressful to navigate. So far people have been reporting back that it's fun to look at. Feedback welcome!

I've also nearly completed the "Intermediate Rails" series, and have learned a lot about how to make Rails apps that are better written so they are more easily extended.

Overall I am very happy with the two courses and feel I've learned a lot in a short while. One great resource that was pointed to in this course was this video of a lecture on SOLID principles by Sandi Metz. She refactors code to use the dependency injection pattern, among other things, so your code is easier to extend and test.

Another thing we learned about that was new to me was using Ruby objects as models by creating classes in your models directory that don't inherit from ActiveRecord. This is a seriously cool way to simplify code and break it down into classes that just do one thing. Classes like that are easier to test and therefore easier to change. Wheeeee!

This is a series of thoughts and ramblings on web development by a web developer and artist who has a passion for learning, puns, and rock climbing.
